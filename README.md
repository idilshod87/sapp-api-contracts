# sapp-api-contracts

Protobuf contract definitions for all Sapp backend services.

## Structure

```
proto/
  sapp/
    common/v1/common.proto          — shared PageInfo / PageCursor types
    messaging/v1/messaging_service.proto  — MessagingService (chat, sync, history)
    media/v1/media_service.proto    — MediaService (upload, processing, retrieval)
```

## Usage

Stubs are **not** generated by consumers. On every push to `main`, CI auto-tags a
semver release and publishes pre-generated packages to GitHub Packages
(`idilshod87/sapp-api-contracts`):

| Consumer | Package | Registry |
|----------|---------|----------|
| Android  | `com.sapp:api-contracts` (Kotlin/Java + gRPC stubs) | Maven — `https://maven.pkg.github.com/idilshod87/sapp-api-contracts` |
| .NET     | `Sapp.ApiContracts` (C# + gRPC stubs) | NuGet — `https://nuget.pkg.github.com/idilshod87/index.json` |

### Android (Gradle)

```kotlin
// settings.gradle.kts — add the GitHub Packages Maven repo (with credentials)
repositories {
    maven {
        url = uri("https://maven.pkg.github.com/idilshod87/sapp-api-contracts")
        credentials {
            username = providers.gradleProperty("gpr.user").orNull ?: System.getenv("GITHUB_ACTOR")
            password = providers.gradleProperty("gpr.token").orNull ?: System.getenv("GITHUB_TOKEN")
        }
    }
}

// build.gradle.kts
dependencies {
    implementation("com.sapp:api-contracts:<version>")
}
```

### .NET (NuGet)

```
dotnet add package Sapp.ApiContracts --version <version>
```

The NuGet source `github` (`https://nuget.pkg.github.com/idilshod87/index.json`)
must be configured with a GitHub token in `NuGet.Config`.

## Development

Install the shared git hooks once per clone:

```
git config core.hooksPath .githooks
```

The `pre-commit` hook mirrors CI (`buf format`, `buf lint`, namespace guardrail)
so proto issues are caught locally. It requires [`buf`](https://buf.build/docs/installation)
in `PATH`. Do not align fields/comments by hand — run `buf format -w` instead.

## Versioning

Package releases are auto-tagged from [Conventional Commits](https://www.conventionalcommits.org/)
on push to `main` (see `.github/workflows/publish.yml`):

- `feat!:` / `BREAKING CHANGE` → major bump
- `feat:` → minor bump
- everything else → patch bump

The proto **package** suffix (`v1` → `v2`) is separate: it changes only on a
wire-breaking redesign so `v1` and `v2` contracts can coexist. Within a `v1`
package, additive changes (new fields, new RPCs) stay backward-compatible and
are enforced by the `buf breaking` gate in `.github/workflows/pr-validate.yml`.
